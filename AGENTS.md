这是 v2ray-agent (vasma) 代理服务内核管理脚本的一个下游修改版本 vasma-chaining。
该版本主要增加了与 vless 链式代理相关的支持，并做了一些其他的小优化。

仓库主要内容就是 install.sh 这一个中文版的管理脚本本体，以及一些附属的配置文件。其他辅助脚本或文档可能过时，最新信息以脚本为准。

更新流程：
我们在最新的 upstream 版本之上重新应用我们 chaining 的改动内容（rebase）来保持更新。
进行合并之前，必须首先阅读 upstream 和 chaining 这两个分支各自先进于共同基础提交的具体改动内容。
然后，详细分析 rebase 时会不会在功能上产生需要解决的问题。
注意，这些问题并不仅仅是 git 显示的冲突，更重要的是还包括上游对某些功能或配置的改动导致我们需要同步修改、上游对某些功能进行了改动导致不兼容因此我们要重做逻辑等问题。
分析完成之后，给出解决方案。助手默认不允许自动执行 rebase 等 git 写入性操作。
